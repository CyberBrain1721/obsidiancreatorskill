
# 典阁

典阁是创作者个人的**永久知识基座**。

它不保存灵感、不保存作品、不保存过程。它只保存一件事：**经过验证、可跨场景长期复用的知识。**

---

## 典阁的职责

灵感属于藏阁。作品属于卷。归档属于归藏。

典阁只收一类东西：**不再依赖特定上下文、能够独立存在的知识资产。**

当你发现某条知识——无论来自创作、技术、法律还是客观事实——反复被引用、持续有效、且不会因为时间推移而失效，它应当进入典阁。

---

## 入典标准

一个东西能不能入典，只问一个问题：

> **一年后，我还需要引用它吗？**

如果可以，入典。如果不确定，留在藏阁。

更具体的判断：

| 维度 | 入典 | 不入典 |
|------|------|--------|
| **创作** | 反复有效的叙事结构、角色设计方法论 | 某个卷的对话草稿、一次性设定 |
| **技术** | 经过验证的代码模式、工具配置规范 | 今天调试时临时用到的解法 |
| **法律** | 关键法律条文、合规要求、行业标准 | 某篇文章对法律的评论 |
| **客观** | 认知偏差清单、经过验证的方法论框架 | 一篇观点文章的摘要 |
| **写作** | 反复使用的模板、经过验证的风格指南 | 某次创作的实验性尝试 |

**核心原则**：典阁存的是「可靠参考」，不是「灵感碎片」。

---

## 典阁结构

```
典阁/
├── 创作体系/
│   ├── 写作方法/       ← 叙事结构、创作方法论
│   ├── 人物模板/       ← 角色设计框架
│   ├── 世界观模板/     ← 世界观构建方法
│   ├── Scene模板/      ← 场景设计模式
│   └── 风格指南/       ← 语言风格、修辞体系
│
├── 技术知识/
│   ├── 代码模式/       ← 可复用的代码片段、架构模式
│   ├── 工具手册/       ← 常用工具的配置与使用方法
│   └── 技术参考/       ← 技术原理、协议规范
│
├── 法律与规则/
│   ├── 法律条文/       ← 原文法条及关键解读
│   ├── 合规要求/       ← 行业合规标准
│   └── 行业标准/       ← 技术标准、行业规范
│
└── 客观知识/
    ├── 概念定义/       ← 学科核心概念的准确界定
    ├── 数据事实/       ← 可验证的客观数据
    └── 方法论框架/     ← 跨学科的分析框架
```

按需生长。不需要的目录可以不创建，需要的可以新增。

典阁不要求你一开始就填满。它随着你的创作和实践，自然沉淀。

---

## 入典的数据流

入典有两条路径：

**路径一：从卷提炼**
```
藏阁 → 共创 → 成卷 → 完成 → 归藏
                            ↓
                    提炼可复用知识 → 入典
```
当你在写卷的过程中反复用到某种方法、某种框架、某种原则，并且验证了它的有效性——卷完成后，从中提炼，入典。

**路径二：直接沉淀**
```
收录 → 判断为客观知识/可靠参考 → 入典
```
当收录的内容本身已经是经过验证的客观规则——比如法律条文、行业标准、技术规范——不需要走完整的卷流程，可以直接入典。

**路径三：主动入库**
```
创作过程中发现/总结的方法 → 入典
```
不需要经过藏阁。你在实践中自己总结出来的方法论，可以直接沉淀进典阁。

---

## 入典指令

对 Agent 说：

```
入典 [内容描述]
```

Agent 会确认：
1. 内容类型（创作/技术/法律/客观）
2. 放在哪个子目录
3. 确认入典后生成典阁条目

示例：
```
你: 入典 从卷003提炼出的"AI学习四门课"方法论框架
你: 入典 《著作权法》第22条合理使用条款
你: 入典 React useEffect 清理函数的正确模式
```

## 典阁与其他层的关系

| 层      | 职责        | 与典阁的关系         |
| ------ | --------- | -------------- |
| **藏阁** | 收录灵感和外部信息 | 典阁的潜在原料来源      |
| **卷**  | 完成具体作品    | 典阁的方法验证场       |
| **归藏** | 归档已完成作品   | 卷进入归藏后，可提炼知识入典 |
| **典阁** | 永久知识基座    | 服务未来的卷和创作      |

```text
藏阁（灵感输入）→ 卷（验证方法）→ 归藏（作品存档）
                                    ↓
                              典阁（方法沉淀）→ 服务下一个卷 ↑
```

此循环永不终止。

---

## 补充说明

典阁不是百科全书。不是你看到什么就存什么。

典阁是**你自己的知识体系**。每一个条目，都经过你的使用、验证、确认。它反映的是你相信什么、你使用什么、你反复依赖什么。

宁可典阁空着，也不要把它变成第二个藏阁。
