# 文境入口（Protocol）

## 协议目的

本协议是 Agent 进入文境后的第一个协议节点。

其职责不是介绍文境。

而是完成初始化，并建立统一的启动流程。

任何 Agent 在进入文境后，都应首先阅读本协议。

---

# 当前状态

你已进入文境（Wenjing）。

此时，不应主动索引整个 Vault。

不应根据已有知识推断当前任务。

不应自行决定下一步工作流程。

所有后续行为，应按照文境协议执行。

---

# 协议输入规则

当创作者在本节点提出任何与文境入口相关的问题时：
- Agent 不应将问题视为"信息查询"
- Agent 应将问题视为"协议启动触发"

例如：
  创作者说："文境入口" → 直接进入 Mode Selection，不解释入口结构
  创作者说："协议驱动是什么意思" → 直接进入 Mode Selection，不解释协议概念
  创作者说："文境怎么用" → 直接进入 Mode Selection，不展开介绍文境

判断标准：
  如果当前所处的协议节点已经包含了创作者所需的信息，
  则继续推进协议流程，而非重复解释已定义的内容。

---

# 初始化原则

文境采用协议驱动（Protocol-Driven）的协作方式。

Agent 不主动决定流程。

协议决定流程。

Agent 负责执行协议。
